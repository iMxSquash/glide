# Security Policy

## Supported versions

Glide n'a pas encore de politique de support multi-versions formelle : seule la dernière version publiée (voir [Releases](https://github.com/iMxSquash/glide/releases)) reçoit des correctifs de sécurité.

| Version | Support |
| ------- | ------- |
| Dernière (`latest`) | ✅ |
| Versions antérieures | ❌ |

## Signaler une vulnérabilité

**Ne jamais ouvrir une issue publique pour une faille de sécurité.**

Utilise l'onglet [Security > Report a vulnerability](https://github.com/iMxSquash/glide/security/advisories/new) de ce repo (GitHub Security Advisories), ce qui permet une divulgation privée et coordonnée. Tu peux aussi envoyer un mail à coussotelwen@gmail.com.

Merci d'inclure :
- une description du problème et de son impact potentiel,
- les étapes pour le reproduire,
- la version de Glide concernée.

## Délai de réponse

Un premier retour est visé sous 7 jours. Le correctif est publié avant toute divulgation publique (divulgation coordonnée) ; merci de nous laisser le temps de le déployer avant de rendre les détails publics.
